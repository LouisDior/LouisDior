Breve introducción


Hoy en día la demanda por el desarrollo de sitios o aplicaciones web ha crecido inconmensurablemente, gracias al acceso y disponibilidad a internet y los múltiples programas o plataformas que permiten el desarrollo de manera inmediata de paginas web, pero hay que tener en cuenta que para un profesional el crear un sitio web conlleva el buen diseño y arquitectura con la que será mostrada el contenido del sitio, además de administrar de manera optima y precisa la información que esta contendrá. A partir de ahí entra la Maquetación web, cuya función no se limita en lo estético sino también en cómo será la fluidez con la que el sitio web interactúa con el usuario. Para ello hay que entender que es, para que sirve, como se debe planificar, cuál es su estándar y como está estructurado, además de conocer lo necesario los beneficios que esta presenta para un sitio web.



¿Qué es la maquetación web?



Se encarga de diseñar y dar forma a la arquitectura con la cual esta estructura un sitio web y todos sus elementos, dando un estilo propio y ordenado para que se vea más atractivo para los usuarios, desde dar contorno a los distintos elementos como títulos, imágenes, archivos multimedia, tablas, texto, entre otros definiendo sus tamaños, color, relleno, márgenes, bordes, divisiones y hasta efectos o animaciones.



¿Para qué sirve?


Nos da la posibilidad de mantener separado y ordenado el contenido de las paginas web que conforman el sitio web. Para hacer más fácil y preciso el desarrollo, mantenimiento y los cambios presentes para él futuro. También, permite tanto a los desarrollados como a los usuarios optimizar el tiempo de respuesta o de búsqueda de información del contenido del sitio web, permitiendo una navegación rápida del sitio.



Planificación del proceso de desarrollo


Algunos puntos o procesos a tener en cuenta para crear o diseñar un sitio web deben ser:

  1. Fijar la zona de trabajo, donde ubicaremos cada uno de los elementos que conformaran el sitio web. Lo cual nos permite mas adelante generar recursos para una rápida navegación.

  2. Separar la presentación del contenido, para reducir el tiempo de desarrollo y de búsqueda del contenido.

  3. Utilizar estándares de desarrollo, para darle forma a la estructura del sitio web.





Estructura básica de la Maquetación web


La estructura básica puede ser presentada de siguiente forma a continuación:


  1. Encabezado (Header): Esta ubicado en la parte superior de la pagina web. Esta conformada por el titulo del sitio web, logo tipo que representa, la foto del usuario, entre otros.

  2. Contenedor: Ocupa la mayor parte del sitio, esta conformado por el contenido de la página, un menú (ya sea lateral o superior), algún tipo de contenido como publicidad.

  3. Menú: Pueda estar ubicado ya sea desde el lado superior debajo del encabezado o de manera lateral (ya sea de izquierda a derecha). Presenta las opciones con las cuales está formado el contenido del sitio web.

  4. Contenido de la página: Se presenta toda la información de las paginas del sitio web, puede ser desde artículos, secciones, párrafos, publicidad, enlaces a otros sitios web o hacia la misma, entre otros.

  5. Pie de página (Footer): Se ubica en la parte inferior de la página. Esta conformada por todas las referencias (bibliografías, licencias, derechos reservados), enlaces a otros sitios web como redes sociales u otras páginas web.


Estándares más utilizados

  1. HTML (HyperText Markup Language): para generar el código con el cual estará estructurado el contenido del sitio web.

  2. CSS (Cascading Stylesheets): para diseñar y definir la forma de la arquitectura y el diseño del sitio web.

  3. JS (JavaScript): para mejorar y programar la interacción de los elementos del sitio web.

Conclusión


En la actualidad debido a la gran demanda que presenta crear sitios web, los desarrolles cuentan con programas que permiten maquetar los sitios web sin necesidad de manejar html, css o javascript, por ejemplo, el editor de texto WordPress que permite ahorra el tiempo de desarrollo para un sitio web, aunque también, la maquetación no se limita con estructurar el código que la conformara, sino que el desarrollador cuenta con herramientas de edición como por ejemplo PhotoShop para crear imágenes e iconos que estarán presentes en el sitio web. Pero queda a criterio de los desarrolladores el cómo crear un sitio web de manera fácil y precisa, y dependerá del funcionamiento que el sitio web deberá presentar.
